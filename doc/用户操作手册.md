## 1. 系统简介

### 1.1 软件概述

Frontend Deploy Manager 是一款专为前端开发者设计的桌面应用程序，用于简化前端项目的构建和部署流程。通过可视化的操作界面，您可以轻松管理多个项目、配置服务器凭证，并一键完成项目的构建与发布。

### 1.2 主要功能

- **项目管理**: 集中管理所有前端项目
- **凭证管理**: 安全存储服务器和 SVN 凭证
- **一键发布**: 支持 SVN、服务器、混合等多种发布方式
- **发布模板**: 保存常用配置，快速发布
- **发布历史**: 查看历史记录和详细日志
- **Node 版本管理**: 自动切换项目所需的 Node 版本

### 1.3 适用场景

- 前端项目日常构建和部署
- 多环境（开发/测试/生产）发布管理
- 团队项目统一部署流程

## 2. 安装与启动

### 2.1 系统要求

| 项目 | 要求 |
|------|------|
| 操作系统 | Windows 10/11 |
| Node.js | 18.0 或更高版本 |
| Git | 需要安装并配置 |
| SVN | 如需 SVN 发布功能需安装 |
| nvm-windows | 如需 Node 版本管理需安装 |

### 2.2 安装步骤

1. 下载安装包（.exe 文件）
2. 双击运行安装程序
3. 选择安装目录
4. 完成安装

### 2.3 首次启动

首次启动时，系统会自动：
- 创建数据存储文件
- 初始化应用配置
- 创建默认数据结构

## 4. 项目管理

### 4.1 添加项目

1. 点击左侧菜单「项目管理」
2. 点击右上角「添加项目」按钮
3. 填写项目信息：

| 字段 | 说明 | 必填 |
|------|------|------|
| 项目名称 | 项目的显示名称 | ✅ |
| 本地路径 | 项目在本地磁盘的路径 | ✅ |
| Git 仓库 | 远程仓库地址（自动获取） | ❌ |
| 默认分支 | 发布时使用的默认分支 | ❌ |
| 构建命令 | 如 `npm run build` | ❌ |
| 产物目录 | 构建输出目录，如 `dist` | ❌ |
| 所属分组 | 项目分组 | ❌ |
| Node 版本 | 项目需要的 Node 版本 | ❌ |
| 项目描述 | 备注信息 | ❌ |

4. 点击「确定」保存

> **提示**: 填写本地路径后，系统会自动扫描 Git 仓库信息

### 4.2 编辑项目

1. 在项目列表中找到目标项目
2. 点击项目卡片上的「编辑」按钮
3. 修改需要更新的信息
4. 点击「确定」保存

### 4.3 删除项目

1. 在项目列表中找到目标项目
2. 点击项目卡片上的「删除」按钮
3. 确认删除操作

> **注意**: 删除项目不会删除发布历史记录

### 4.4 项目分组

通过分组功能可以更好地组织项目：

1. 在添加/编辑项目时选择所属分组
2. 在设置页面可以管理分组（添加、编辑、删除）

## 5. 凭证管理

### 5.1 服务器凭证

#### 添加服务器凭证

1. 点击左侧菜单「服务器凭证」
2. 点击「添加凭证」按钮
3. 填写凭证信息：

| 字段 | 说明 | 必填 |
|------|------|------|
| 凭证名称 | 用于识别的名称 | ✅ |
| 服务器地址 | IP 地址或域名 | ✅ |
| SSH 端口 | 默认 22 | ✅ |
| 用户名 | SSH 登录用户名 | ✅ |
| 认证方式 | 密码 或 私钥 | ✅ |
| 密码/私钥 | 根据认证方式填写 | ✅ |
| 私钥密码 | 如果私钥有密码 | ❌ |
| 环境 | dev/test/prod | ❌ |
| 描述 | 备注信息 | ❌ |

4. 点击「测试连接」验证凭证是否正确
5. 点击「确定」保存

#### 认证方式说明

- **密码认证**: 直接输入 SSH 登录密码
- **私钥认证**: 粘贴私钥内容，如果私钥有密码保护，需填写私钥密码

### 5.2 SVN 凭证

#### 添加 SVN 凭证

1. 点击左侧菜单「SVN 凭证」
2. 点击「添加凭证」按钮
3. 填写凭证信息：

| 字段 | 说明 | 必填 |
|------|------|------|
| 凭证名称 | 用于识别的名称 | ✅ |
| SVN 地址 | SVN 仓库 URL | ✅ |
| 用户名 | SVN 登录用户名 | ✅ |
| 密码 | SVN 登录密码 | ✅ |
| 环境 | dev/test/prod | ❌ |
| 描述 | 备注信息 | ❌ |

4. 点击「测试连接」验证凭证是否正确
5. 点击「确定」保存

### 5.3 环境分类

凭证可以按环境分类管理：
- **dev**: 开发环境
- **test**: 测试环境
- **prod**: 生产环境

通过标签颜色区分不同环境的凭证。

## 6. 发布操作

### 6.1 进入发布页面

1. 点击左侧菜单「发布」
2. 系统会显示发布配置界面

### 6.2 选择项目

1. 在「选择项目」下拉框中选择要发布的项目
2. 系统会自动加载项目的默认配置

### 6.3 选择分支

1. 在「选择分支」下拉框中选择要发布的分支
2. 下拉框会显示项目的所有分支

### 6.4 选择发布类型

根据需要选择发布类型：

#### SVN 发布

适用于将构建产物发布到 SVN 服务器。

配置项：
| 字段 | 说明 |
|------|------|
| SVN 凭证 | 选择已配置的 SVN 凭证 |
| SVN 路径 | 发布目标路径 |
| 提交信息 | SVN 提交说明 |
| 备份旧版本 | 是否备份现有文件 |

#### 服务器发布

适用于将构建产物通过 SFTP 上传到远程服务器。

配置项：
| 字段 | 说明 |
|------|------|
| 服务器凭证 | 选择已配置的服务器凭证 |
| 远程路径 | 服务器上的目标路径 |
| 备份远程目录 | 是否备份现有目录 |

#### 混合发布

同时发布到多个目标（SVN + 多个服务器）。

配置项：
1. 点击「添加发布目标」
2. 选择目标类型（SVN 或服务器）
3. 配置对应的凭证和路径
4. 可添加多个目标

### 6.5 高级选项

| 选项 | 说明 |
|------|------|
| 需要构建 | 是否执行构建命令（取消则使用现有产物） |
| 备份 | 发布前备份目标目录 |

### 6.6 执行发布

1. 配置完成后，点击「开始发布」按钮
2. 系统会显示发布进度面板，实时展示：
   - 当前执行步骤
   - 构建日志
   - 上传进度
   - 错误信息（如有）

### 6.7 取消发布

发布过程中，如需取消：
1. 点击进度面板中的「取消发布」按钮
2. 系统会安全终止当前操作

### 6.8 发布状态

| 状态 | 说明 |
|------|------|
| 构建中 | 正在执行构建命令 |
| 上传中 | 正在上传文件到目标 |
| 已完成 | 发布成功完成 |
| 失败 | 发布过程中出错 |
| 已取消 | 用户手动取消发布 |

## 7. 发布模板

### 7.1 创建模板

发布模板可以保存常用的发布配置，方便快速发布。

1. 点击左侧菜单「发布模板」
2. 点击「创建模板」按钮
3. 填写模板信息：

| 字段 | 说明 |
|------|------|
| 模板名称 | 模板的显示名称 |
| 关联项目 | 选择关联的项目 |
| 发布类型 | SVN/服务器/混合 |
| 凭证配置 | 根据发布类型配置 |
| 目标路径 | 发布目标路径 |
| 其他选项 | 备份、前后置命令等 |

4. 点击「确定」保存

### 7.2 使用模板

1. 在「发布模板」页面找到目标模板
2. 点击模板卡片上的「使用」按钮
3. 系统会自动跳转到发布页面并加载模板配置
4. 确认配置后点击「开始发布」

### 7.3 管理模板

- **编辑**: 修改模板配置
- **删除**: 删除不需要的模板
- **筛选**: 按项目筛选模板列表

## 8. 发布历史

### 8.1 查看历史记录

1. 点击左侧菜单「发布历史」
2. 系统显示所有发布记录列表

历史记录包含：
| 字段 | 说明 |
|------|------|
| 项目名称 | 发布的项目 |
| 发布类型 | SVN/服务器/混合 |
| 分支 | 发布的 Git 分支 |
| Commit | Git 提交 hash |
| 状态 | 发布结果状态 |
| 开始时间 | 发布开始时间 |
| 完成时间 | 发布完成时间 |

### 8.2 筛选记录

1. 使用页面上方的项目下拉框
2. 选择特定项目查看该项目的发布历史
3. 选择「全部项目」查看所有记录

### 8.3 查看日志

1. 找到目标历史记录
2. 点击「查看日志」按钮
3. 弹出窗口显示详细日志信息：
   - 发布状态标签
   - 分支和 Commit 信息
   - 发布时间
   - 带时间戳的操作日志

### 8.4 复制日志

在日志查看窗口：
1. 点击「复制日志」按钮
2. 日志内容会复制到系统剪贴板

## 9. 系统设置

### 9.1 进入设置

点击左侧菜单「设置」

### 9.2 配置导出

将系统配置导出为 JSON 文件：

1. 点击「导出配置」按钮
2. 选择保存位置和文件名
3. 选择是否包含密码
4. 点击「保存」

> **注意**: 包含密码的配置文件请妥善保管

从 JSON 文件导入配置：

1. 点击「导入配置」按钮
2. 选择配置文件
3. 系统会自动导入：
   - 项目列表
   - 分组信息
   - 服务器凭证
   - SVN 凭证
   - 发布模板

### 9.4 清空数据

清空所有应用数据：

1. 点击「清空所有数据」按钮
2. 确认操作

> **警告**: 此操作不可恢复，请谨慎操作

### 9.5 数据目录

点击「打开数据目录」可以直接访问应用数据存储位置。

## 10. 常见问题

### 10.1 项目相关

**Q: 添加项目时提示"无法获取 Git 信息"?**

A: 请确保：
- 项目路径正确
- 项目是一个有效的 Git 仓库
- Git 已正确安装并配置

**Q: 项目配置的 Node 版本不生效?**

A: 请确保：
- 已安装 nvm-windows
- 配置的 Node 版本已通过 nvm 安装
- NVM_HOME 环境变量已正确设置

### 10.2 凭证相关

**Q: 服务器连接测试失败?**

A: 请检查：
- 服务器地址和端口是否正确
- 用户名和密码是否正确
- 服务器 SSH 服务是否正常运行
- 网络是否可达

**Q: 使用私钥认证失败?**

A: 请检查：
- 私钥格式是否正确（通常是 -----BEGIN 开头）
- 私钥是否有密码保护
- 服务器是否配置了对应的公钥

### 10.3 发布相关

**Q: 构建失败?**

A: 请检查：
- 构建命令是否正确
- 项目依赖是否已安装（npm install）
- Node 版本是否满足要求

**Q: 上传速度很慢?**

A: 系统默认使用压缩上传方式。如果仍然较慢：
- 检查网络连接
- 检查服务器带宽
- 考虑减少构建产物大小

**Q: 发布过程中想取消?**

A: 点击发布进度面板中的「取消发布」按钮即可安全终止。

### 10.4 其他问题

**Q: 数据存储在哪里?**

A: 数据存储在用户数据目录下的 JSON 文件中。可通过「设置」->「打开数据目录」访问。

**Q: 如何备份配置?**

A: 使用「设置」->「导出配置」功能，将配置导出为 JSON 文件保存。

**Q: 忘记密码怎么办?**

A: 服务器和 SVN 凭证的密码是加密存储的，无法找回。请重新配置凭证。
