# Node.js 版本治理操作文档

## 功能概述

本项目支持为每个项目声明所需的 Node.js 版本，在构建/发布时自动校验并切换版本，确保不同项目在正确的 Node 环境下构建。

## 1. 为项目声明 Node 版本

1. 进入「项目管理」页面
2. 新建或编辑项目
3. 在表单中找到 **「Node 版本」** 字段，输入版本号（如 `16.20.2`、`18.17.0`）
4. 留空则表示不进行版本校验，使用当前系统默认 Node 版本

> 版本号无需带 `v` 前缀，系统会自动补全。

## 2. 版本校验流程

开启「需要构建」开关并执行发布时，系统会自动执行以下检查：

```
┌─────────────────────────────┐
│ 读取项目声明的 Node 版本      │
│ (nodeVersion 字段)           │
└──────────┬──────────────────┘
           │
           ▼
┌─────────────────────────────┐
│ 执行 node -v 获取当前版本    │
└──────────┬──────────────────┘
           │
           ▼
┌─────────────────────────────┐
│ 版本一致？                   │
│ YES → 放行，继续构建         │
│ NO  → 进入 nvm 检测         │
└──────────┬──────────────────┘
           │
           ▼
┌─────────────────────────────┐
│ nvm 已安装？                 │
│ NO  → 报错终止               │
│ YES → 检查版本安装状态       │
└──────────┬──────────────────┘
           │
           ▼
┌─────────────────────────────┐
│ 版本已安装？                 │
│ YES → 弹窗确认切换           │
│ NO  → 弹窗确认安装并切换     │
└─────────────────────────────┘
```

## 3. 前置条件

### Windows 系统
- 安装 [nvm-windows](https://github.com/coreybutler/nvm-windows/releases)
- 下载 `nvm-setup.exe` 并安装
- 安装后在命令行验证：`nvm version`

### macOS/Linux 系统
- 安装 [nvm](https://github.com/nvm-sh/nvm)
- 执行：`curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.7/install.sh | bash`
- 重启终端后验证：`nvm --version`

## 4. 常见错误排查

| 错误信息 | 原因 | 解决方案 |
|---------|------|---------|
| `本项目依赖 Node vx.xx.x，请先安装 nvm 后再试` | 系统未安装 nvm | 按上方指引安装 nvm |
| `无法获取当前 Node 版本` | Node 未安装或不在 PATH 中 | 确认 `node -v` 可正常执行 |
| `切换 Node 版本失败` | nvm use 命令执行失败 | 手动执行 `nvm use <version>` 检查原因 |
| `安装 Node vx.xx.x 失败` | 网络问题或版本号不存在 | 检查网络，确认版本号正确（到 [nodejs.org](https://nodejs.org) 查看） |
| `用户主动取消版本切换` | 用户点击了取消按钮 | 重新发布并在弹窗中点击确认 |
| `用户拒绝自动安装` | 用户取消了安装确认 | 手动执行 `nvm install <version>` 后重新发布 |

## 5. 注意事项

- 版本切换是**全局生效**的，切换后会影响同一台机器上的所有 Node 进程
- 如果同时有多个项目在构建，版本切换可能产生冲突，建议串行执行
- 版本号必须精确匹配（如 `16.20.2`），不支持范围版本（如 `^16.0.0`）
- nvm install 需要网络连接，下载大版本 Node 可能需要几分钟时间
