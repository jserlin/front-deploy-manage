# 前端项目构建与部署管理系统 - 系统设计文档

## 1. 系统概述

### 1.1 项目名称
**Frontend Deploy Manager** (前端部署管理器)

### 1.2 系统定位
一款基于 Electron + Vue3 的桌面应用，为前端开发者提供可视化的项目构建与部署管理工具，支持 SVN 发布、服务器发布及混合发布流程。

### 1.3 核心价值
- 统一管理多个前端项目的构建与部署流程
- 可视化配置服务器和 SVN 凭证
- 支持一键发布到多个环境
- 提供发布历史追溯和日志查看
- 配置可导出共享，便于团队协作
- 支持 Node 版本管理（通过 nvm-windows）

---

## 2. 技术栈选型

### 2.1 核心框架
- **主框架**: Electron 28+
- **前端框架**: Vue 3.4+ (Composition API)
- **开发语言**: TypeScript 5.0+
- **构建工具**: Vite 5.0+

### 2.2 UI 组件库
- **组件库**: Element Plus 2.5+
- **图标**: @element-plus/icons-vue
- **样式**: SCSS + CSS Variables

### 2.3 状态管理与路由
- **状态管理**: Pinia 2.1+
- **路由**: Vue Router 4.2+

### 2.4 数据存储
- **本地数据库**: JSON 文件存储（轻量级实现）
- **配置文件**: JSON (用于导入导出)
- **加密**: Node.js crypto 模块 (AES-256-CBC)

### 2.5 核心功能库
- **Git 操作**: simple-git
- **SVN 操作**: child_process 调用 svn 命令行
- **SSH/SFTP**: ssh2
- **文件操作**: fs-extra
- **构建工具集成**: 
  - npm/yarn/pnpm (通过 child_process)
  - 支持自定义构建命令
  - Node 版本通过 PATH 注入方式切换
- **压缩上传**: Windows 内置 tar.exe

### 2.6 辅助工具库
- **日志**: console + electron-log
- **进程通信**: electron ipcMain/ipcRenderer
- **进度显示**: ssh2 SFTP 传输进度
- **时间处理**: dayjs

### 5.6 Node 版本管理模块

#### 5.6.1 实现方案
采用 **PATH 注入方式**，而非全局切换 Node 版本：

- 检测 nvm-windows 是否可用
- 检测目标 Node 版本是否已安装
- 构建时将目标 Node 版本路径注入 PATH 环境变量
- 不直接调用 nvm.exe，避免终端检测问题

#### 5.6.2 核心代码
```typescript
// build.service.ts
private getNodeEnvPath(nodeVersion?: string): { [key: string]: string | undefined } {
  if (!nodeVersion) return { ...process.env, NODE_ENV: 'production' }
  const nvmHome = process.env.NVM_HOME
  if (!nvmHome) return { ...process.env, NODE_ENV: 'production' }
  const nodeDir = path.join(nvmHome, `v${nodeVersion}`)
  if (!fs.existsSync(nodeDir)) return { ...process.env, NODE_ENV: 'production' }
  const newPath = `${nodeDir};${process.env.PATH}`
  return { ...process.env, PATH: newPath, NODE_ENV: 'production' }
}
```
