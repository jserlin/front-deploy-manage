# 前端项目构建与部署管理系统 - 系统设计文档

## 1. 系统概述

### 1.1 项目名称
**Frontend Deploy Manager** (前端部署管理器)

### 1.2 系统定位
一款基于 Electron + Vue3 的桌面应用，为前端开发者提供可视化的项目构建与部署管理工具，支持 SVN 发布、服务器发布及混合发布流程。

### 1.3 核心价值
- 统一管理多个前端项目的构建与部署流程
- 可视化配置服务器和 SVN 凭证
- 支持一键发布到多个环境
- 提供发布历史追溯和回滚能力
- 配置可导出共享，便于团队协作

## 2. 技术栈选型

### 2.1 核心框架
- **主框架**: Electron 28+
- **前端框架**: Vue 3.4+ (Composition API)
- **开发语言**: TypeScript 5.0+
- **构建工具**: Vite 5.0+

### 2.2 UI 组件库
- **组件库**: Element Plus 2.5+
- **图标**: @element-plus/icons-vue
- **样式**: SCSS + CSS Variables

### 2.3 状态管理与路由
- **状态管理**: Pinia 2.1+
- **路由**: Vue Router 4.2+

### 2.4 数据存储
- **本地数据库**: SQLite3 (better-sqlite3)
- **配置文件**: JSON (用于导入导出)
- **加密**: Node.js crypto 模块 (AES-256)

### 2.5 核心功能库
- **Git 操作**: simple-git
- **SVN 操作**: svn-node-wrapper (或通过 child_process 调用 svn 命令行)
- **SSH/SFTP**: ssh2
- **文件操作**: fs-extra, chokidar (文件监听)
- **构建工具集成**: 
  - npm/yarn/pnpm (通过 child_process)
  - 支持自定义构建命令
- **进程管理**: child_process, tree-kill

### 2.6 辅助工具库
- **日志**: electron-log
- **进程通信**: electron ipcMain/ipcRenderer
- **进度显示**: ssh2-streams (SFTP 传输进度)
- **时间处理**: dayjs

## 4. 数据库设计

### 4.1 数据表结构

#### 4.1.1 项目表 (projects)
```sql
CREATE TABLE projects (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  name TEXT NOT NULL,                    -- 项目名称
  local_path TEXT NOT NULL UNIQUE,       -- 本地路径
  git_repo TEXT,                         -- Git 仓库地址
  git_branch TEXT DEFAULT 'main',        -- 当前分支
  build_command TEXT DEFAULT 'npm run build', -- 构建命令
  output_dir TEXT DEFAULT 'dist',        -- 构建产物目录
  group_id INTEGER,                      -- 所属分组
  description TEXT,                      -- 项目描述
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (group_id) REFERENCES groups(id)
);
```

#### 4.1.2 项目分组表 (groups)
```sql
CREATE TABLE groups (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  name TEXT NOT NULL UNIQUE,             -- 分组名称
  color TEXT DEFAULT '#409EFF',          -- 分组颜色
  sort_order INTEGER DEFAULT 0,          -- 排序
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

#### 4.1.3 服务器凭证表 (server_credentials)
```sql
CREATE TABLE server_credentials (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  name TEXT NOT NULL,                    -- 凭证名称
  host TEXT NOT NULL,                    -- 服务器IP
  port INTEGER DEFAULT 22,               -- SSH端口
  username TEXT NOT NULL,                -- 用户名
  auth_type TEXT DEFAULT 'password',     -- 认证方式: password/key
  password TEXT,                         -- 加密存储的密码
  private_key TEXT,                      -- 加密存储的私钥
  passphrase TEXT,                       -- 加密存储的私钥密码
  environment TEXT DEFAULT 'dev',        -- 环境: dev/test/prod
  description TEXT,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

#### 4.1.4 SVN 凭证表 (svn_credentials)
```sql
CREATE TABLE svn_credentials (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  name TEXT NOT NULL,                    -- 凭证名称
  svn_url TEXT NOT NULL,                 -- SVN 地址
  username TEXT NOT NULL,                -- 用户名
  password TEXT NOT NULL,                -- 加密存储的密码
  environment TEXT DEFAULT 'dev',        -- 环境
  description TEXT,
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  updated_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

#### 4.1.5 发布模板表 (deploy_templates)
```sql
CREATE TABLE deploy_templates (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  name TEXT NOT NULL,                    -- 模板名称
  project_id INTEGER,                    -- 关联项目
  deploy_type TEXT NOT NULL,             -- svn/server/mixed
  server_credential_id INTEGER,          -- 服务器凭证ID
  svn_credential_id INTEGER,             -- SVN凭证ID
  remote_path TEXT,                      -- 远程路径
  svn_path TEXT,                         -- SVN路径
  backup_enabled INTEGER DEFAULT 1,      -- 是否备份
  pre_command TEXT,                      -- 构建前命令
  post_command TEXT,                     -- 构建后命令
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (project_id) REFERENCES projects(id),
  FOREIGN KEY (server_credential_id) REFERENCES server_credentials(id),
  FOREIGN KEY (svn_credential_id) REFERENCES svn_credentials(id)
);
```

#### 4.1.6 发布历史表 (deploy_history)
```sql
CREATE TABLE deploy_history (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  project_id INTEGER NOT NULL,           -- 项目ID
  template_id INTEGER,                   -- 模板ID
  deploy_type TEXT NOT NULL,             -- 发布类型
  git_branch TEXT,                       -- 发布分支
  git_commit TEXT,                       -- 发布commit
  status TEXT DEFAULT 'pending',         -- 状态: pending/building/uploading/success/failed
  log TEXT,                              -- 发布日志
  started_at DATETIME,                   -- 开始时间
  finished_at DATETIME,                  -- 完成时间
  created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
  FOREIGN KEY (project_id) REFERENCES projects(id)
);
```

## 5. 核心模块设计

### 5.1 项目管理模块

#### 5.1.1 功能列表
- 项目列表展示（卡片/列表视图）
- 添加项目（扫描本地 Git 项目）
- 编辑项目信息
- 删除项目
- 项目分组管理
- 项目搜索与筛选
- 查看项目详情（Git 状态、分支信息）

#### 5.1.2 交互流程

**添加项目流程**:
1. 用户点击"添加项目"
2. 选择本地项目目录
3. 系统自动检测是否为 Git 项目
4. 自动读取 Git 仓库信息、当前分支
5. 用户填写项目名称、构建命令、产物目录
6. 选择所属分组
7. 保存到数据库

**编辑项目流程**:
1. 点击项目卡片"编辑"按钮
2. 修改项目信息
3. 保存更新

**查看项目详情**:
- 显示 Git 状态（分支、提交记录、是否有未提交更改）
- 显示最近发布记录
- 显示发布模板列表

#### 5.1.3 数据模型

#### 5.2.1 功能列表

**服务器凭证**:
- 添加服务器凭证
- 编辑凭证信息
- 测试连接（SSH 连接测试）

**SVN 凭证**:
- 添加 SVN 凭证

- 测试连接（SVN info 命令测试）
- 删除凭证
- 按环境分类展示

#### 5.2.2 密码加密方案
- 使用 AES-256-CBC 加密算法
- 加密密钥存储在 Electron safeStorage 中
- 加密流程:
  1. 用户输入密码
  2. 从 safeStorage 获取加密密钥
  3. 使用 crypto.createCipheriv 加密
  4. 存储加密后的密文到数据库
- 解密流程相反

### 5.3 SVN 发布功能模块

#### 5.3.1 功能列表
- 选择项目和 Git 分支
- 配置构建命令
- 配置 SVN 目标路径
- 执行构建
- 发布到 SVN
- 查看发布进度和日志
- 发布前备份
- 自定义提交信息

#### 5.3.2 发布流程

```
开始
  ↓
选择项目和分支
  ↓
拉取最新代码 (git pull)
  ↓
切换到目标分支
  ↓
执行构建前命令 (可选)
  ↓
执行构建命令 (npm run build)
  ↓
构建成功?
  ├─ 否 → 记录错误日志 → 结束
  └─ 是 ↓
备份 SVN 旧版本 (可选)
  ↓
上传构建产物到 SVN 临时目录
  ↓
提交到 SVN (svn commit)
  ↓
记录发布历史
  ↓
执行构建后命令 (可选)
  ↓
结束
```

#### 5.3.3 SVN 路径模板配置
支持变量替换:
- `{projectName}`: 项目名称
- `{branch}`: 分支名称
- `{date}`: 日期 (YYYYMMDD)
- `{time}`: 时间 (HHmmss)
- `{env}`: 环境名称

示例:
```
https://svn.example.com/projects/{projectName}/{env}/{date}
```

### 5.5 混合发布流程模块

#### 5.5.1 功能列表
- 单次构建多次发布
- 配置发布目标（SVN + 多个服务器）
- 配置发布顺序
- 统一的发布进度展示
- 发布失败回滚

#### 5.5.2 发布流程

```
开始
  ↓
选择项目和分支
  ↓
拉取代码 → 切换分支
  ↓
执行构建 (只构建一次)
  ↓
构建成功?
  ├─ 否 → 结束
  └─ 是 ↓
┌──────────────────────────┐
│  并行或串行发布          │
│  ├─ SVN 发布 (任务1)     │
│  ├─ 服务器1发布 (任务2)  │
│  └─ 服务器2发布 (任务3)  │
└──────────────────────────┘
  ↓
全部成功?
  ├─ 是 → 记录成功日志 → 结束
  └─ 否 → 记录失败日志 → 结束
```

#### 5.5.3 任务编排界面
- 拖拽排序发布顺序
- 支持并行发布配置
- 实时显示每个任务状态
- 失败任务可单独重试

## 14. 风险与挑战

### 14.1 技术风险
- **跨平台兼容性**: Windows/macOS/Linux 文件路径差异
  - 解决方案: 使用 path 模块统一处理路径
- **SVN 命令行依赖**: 需要用户安装 SVN 客户端
  - 解决方案: 提供检测和安装引导
- **SSH 连接稳定性**: 网络不稳定可能导致上传中断
  - 解决方案: 实现断点续传和重试机制

### 14.2 用户体验风险
- **构建时间过长**: 用户可能误以为程序卡死
  - 解决方案: 提供实时日志和进度反馈
- **配置复杂度**: 新用户可能不知如何配置
  - 解决方案: 提供配置向导和默认模板
